WiFi 6 (802.11ax) 两种Sounding + 下行MU-MIMO 交互过程对比

WiFi 6 两种Sounding方式与下行MU-MIMO交互对比

引言

在WiFi 6 (802.11ax) 带来的众多技术革新中，多用户多输入多输出 (MU-MIMO) 无疑是提升网络容量的关键技术之一。它允许AP在同一时频资源上同时与多个STA通信，就像在一条高速公路上开辟了多条车道，大大提高了通行效率。

但要实现高效的MU-MIMO传输，有一个环节至关重要，那就是Sounding（信道探测）。AP需要知道每个STA的信道状态信息，才能计算出合适的预编码矩阵，让多用户数据流”各行其道”，互不干扰。

今天，笔者就和大家深入聊聊WiFi 6中两种不同的Sounding方式——批量式（并行）和顺序式（逐次），看看它们各自的工作原理、优缺点，以及在实际系统中的考量。

方式一：批量式Sounding（并行探测）

批量式Sounding的核心思想是：一次公告，同时测量，依次反馈。AP先让所有STA做好准备，然后发送一次探测信号，大家一起测量信道，最后再挨个把测量结果反馈回来。

完整交互时序图

流程特点

  1. 统一公告：AP先一次性给所有参与探测的STA发送NDPA公告，告诉大家”准备好，我们要开始测信道了”
  2. 同时测量：AP发送一次NDP，所有STA同时测量下行信道，这样大家的信道测量时刻是一致的
  3. 依次反馈：AP通过Trigger帧逐个触发STA发送CBF反馈，收集所有STA的信道信息
  4. 统一传输：收集完所有信道信息后，AP进行预编码，统一进行下行MU-MIMO传输

这种方式的优势很明显：信道测量时间点一致，能保证AP获得的多用户信道信息是”同步”的，这对MU-MIMO的性能至关重要。

方式二：顺序式Sounding（逐次探测）

顺序式Sounding则是另一种思路：逐个完成，依次推进。AP先和第一个STA完成完整的探测流程，搞定之后再和第二个STA重复同样的步骤，最后才进行MU-MIMO传输。

完整交互时序图

流程特点

  1. 逐个处理：AP先与第一个STA完成完整探测流程：NDPA → NDP → 测量 → Trigger → CBF → 存储
  2. 依次推进：第一个STA完成后，AP再与第二个STA重复同样的完整流程
  3. 统一传输：所有STA探测完成后，AP统一进行下行MU-MIMO传输

这种方式的好处是实现简单，AP不需要协调多个STA同时测量，逐个处理逻辑清晰。但正如我们后面会分析的，它也带来了一些性能上的代价。

两种Sounding方式详细对比

为了更直观地看出两种方式的差异，我们从流程和性能两个维度来做个对比。

1. 流程对比

对比项 批量式Sounding (并行探测) 顺序式Sounding (逐次探测)
NDPA发送 AP一次性给所有参与STA发送NDPA AP逐个给每个STA发送NDPA
NDP发送 AP一次性发送NDP，所有STA同时测量 AP逐个给每个STA发送NDP
信道测量时间 所有STA在同一时间点测量信道 不同STA在不同时间点测量信道
CBF反馈 AP依次触发各个STA轮流反馈 每个STA测量完立即反馈
总交互帧数 较少（2个STA时：NDPA×1 + NDP×1 + Trigger×2 + CBF×2） 较多（2个STA时：每个STA都需要NDPA + NDP + Trigger + CBF）

这里需要说明一下，虽然表格中都写了”NDPA×2”，但在实际的批量式Sounding中，AP通常可以用一个多用户NDPA（广播或组播形式）一次性通知所有STA，而不需要真的发送两次独立的NDPA帧。这也是批量式协议开销更低的原因之一。

2. 性能对比

对比项 批量式Sounding 顺序式Sounding
时间效率 较高，NDP只需发送一次，所有STA同时测量 较低，每个STA都需要独立发送NDPA+NDP
信道一致性 好，所有STA信道测量在同一时间点完成 差，不同STA测量存在时间差
协议开销 较低，控制帧数量少 较高，控制帧数量多
实现复杂度 略高，需要调度多个STA同时测量 较低，逐个处理简单

从这张对比表可以看出，批量式Sounding在性能上明显占优，特别是信道一致性和时间效率这两个关键指标。而顺序式Sounding唯一的优势就是实现简单。

顺序式Sounding的缺点分析

虽然顺序式Sounding实现简单，但在实际系统中，它的缺点也非常明显，特别是对MU-MIMO性能的影响不容忽视。

1. 信道测量时间不同步，导致MU-MIMO性能损失

这是顺序式Sounding最致命的缺点。试想一下：

  • STA1在t1时刻测量了信道，得到H1
  • 然后AP又花了时间去和STA2完成探测，STA2在t2时刻测量了信道，得到H2
  • 当AP终于收集完所有信息，开始MU-MIMO传输时，已经是t3时刻了

在t1到t3这段时间里，无线信道可能已经发生了变化——STA可能移动了，周围环境可能变了，H1已经不是t3时刻的真实信道了。

这种信道老化（Channel Aging） 会导致AP计算的预编码矩阵不准确，造成多用户间干扰增加，严重降低MU-MIMO的解调性能。在高速移动场景下，这个问题会更加突出。

2. 交互次数多，协议开销大

对于K个STA的场景：

  • 批量式：只需要发送1次NDPA公告（多用户形式）和1次NDP，总共2次传输，再加上K次Trigger和K次CBF
  • 顺序式：需要发送K次NDPA和K次NDP，总共2K次传输，再加上K次Trigger和K次CBF

当STA数量增加时，顺序式的控制帧数量会线性增长，占用大量宝贵的空口资源，降低有效数据传输时间。

3. 总探测时间更长，信道老化更严重

顺序处理每个STA的完整流程，意味着总探测时间比批量式更长。更糟糕的是，第一个STA完成探测到最终MU-MIMO传输之间的时间间隔更大，进一步加剧了信道信息老化问题。

这就形成了一个恶性循环：探测时间长 → 信道老化严重 → MU-MIMO性能差 → 可能需要更频繁的探测 → 开销更大。

4. 不支持多用户同时探测，扩展性差

当参与MU-MIMO的STA数量增加时，顺序式的总探测时间会线性增长。而批量式在增加STA数量时，NDPA和NDP不需要增加发送次数，只需要增加轮询反馈的次数即可。

在WiFi 6支持最多8个下行MU-MIMO用户的情况下，顺序式的扩展性问题会变得非常明显。

各帧作用总览

在深入了解了两种Sounding方式后，我们再来回顾一下整个交互过程中涉及的关键帧类型，帮助大家更好地理解每个环节的作用。

帧类型 发送方 接收方 作用
NDPA (Null Data Packet Announcement) AP STA 空数据包公告。通告目标STA即将开始Sounding过程，让STA做好信道测量准备。
NDP (Null Data Packet) AP STA 空数据包。只包含L-LTF等训练前导码，不携带数据净荷。目标STA利用前导码测量下行信道，得到信道矩阵。
Trigger Frame (TB PPDU) AP 单个STA 触发帧。为目标STA分配上行资源（RU，Resource Unit），触发该STA发送CBF反馈。WiFi 6中使用基于触发的上行传输。
CBF (Compressed Beamforming Feedback) STA AP 压缩波束成形反馈。STA将测量得到的下行信道矩阵进行量化压缩，反馈给AP。AP基于这些信息计算预编码矩阵。
MU-MIMO PPDU AP 多个STA 多用户下行物理层协议数据单元。AP利用收集到的各个STA信道信息，对不同STA的数据进行预编码，在相同的时频资源上同时发送多个数据流，实现空间复用。
ACK STA AP 确认帧。每个STA成功接收数据后，向AP发送确认。

笔者的实践视角与思考

作为一名通信IC设计者，笔者在实际工作中接触过不少WiFi芯片的设计和验证，在这里想分享一些对Sounding机制的实践视角思考。

为什么信道一致性如此重要？

在MU-MIMO系统中，预编码矩阵的设计是基于所有用户的信道矩阵联合计算的。如果不同用户的信道测量时间不一致，就像用不同时刻拍摄的照片来合成一幅全景图，结果肯定会有”错位”。

这种错位导致的后果是：本来应该被”零陷”掉的用户间干扰没有被完全消除，而本来应该增强的目标信号强度也打了折扣。在实际系统测试中，我们经常能看到，信道老化带来的性能损失可能会让MU-MIMO的增益大打折扣，甚至不如SU-MIMO。

批量式的实现真的很复杂吗？

其实，从协议实现的角度来看，批量式Sounding并没有想象中那么复杂。AP只需要在NDPA中携带参与探测的STA列表，然后发送一次NDP，之后按照预定的顺序依次触发各个STA反馈即可。

真正的挑战在于：如何处理部分STA测量失败的情况，以及如何优化Trigger帧的调度，让反馈过程更高效。但这些都是工程实现上可以解决的问题，与带来的性能收益相比，这些复杂度是值得的。

Sounding周期的选择是一门艺术

在实际系统中，Sounding不会只做一次，而是会周期性进行。周期的选择是一个需要权衡的问题：

  • 周期太短：协议开销大，有效数据传输时间减少
  • 周期太长：信道信息老化，MU-MIMO性能下降

这个周期需要根据实际应用场景动态调整：在静止或低速移动场景下，周期可以长一些；在高速移动场景下，周期需要相应缩短。有些高级的WiFi芯片还支持根据信道变化情况自适应调整Sounding周期。

总结

通过以上的分析，我们可以得出以下几点核心结论：

  1. 批量式Sounding是MU-MIMO的首选：它能提供更好的信道一致性和更高的时间效率，是实现高效MU-MIMO传输的理想选择。

  2. 顺序式Sounding仅适用于特定场景：虽然实现简单，但信道不一致和开销大的缺点使其在大多数场景下不如批量式。

  3. 信道一致性是MU-MIMO性能的关键：所有STA的信道信息最好在同一时间点获取，这样才能保证预编码矩阵的准确性。

  4. Sounding周期需要精心设计：实际系统中需要根据场景动态调整Sounding周期，在开销和信道跟踪性能之间取得平衡。

WiFi 6的MU-MIMO技术为我们带来了网络容量的巨大提升，而Sounding机制正是这一切的基础。理解两种Sounding方式的差异，有助于我们更好地设计和优化WiFi系统，让MU-MIMO的性能得到充分发挥。
